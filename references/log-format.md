# Distill Log Format Specification

This file describes the distill log format generated by `record.py`, for reference when producing output manually in the Web environment.

## File Naming

```
YYYY-MM-DD_HH-MM_<topic>.md
```

Examples:
- `2026-04-16_14-30_refactor-auth-flow.md`
- `2026-04-15_09-12_debug-memory-leak.md`
- `2026-04-14_21-05_design-review.md`

Topic naming rules:
- Hyphen-separated, lowercase
- 1–4 words reflecting the core subject of the conversation
- Prefer specific terms (`cuda-memory-opt`) over generic ones (`coding-help`)

---

## File Structure

```markdown
---
date: YYYY-MM-DD HH:MM
topic: <topic>
turns: <number of turns>
sentiment: neutral | positive | negative
reason: <optional sentiment reason>
tags: []
---

# <topic> — <date>

## Metadata
- Total turns: N
- Session file: <jsonl-filename>.jsonl
- Reason: <reason, only present when provided>

---

## Conversation

### Turn 1

**User:**
(verbatim user text, preserved character for character)

**Assistant:**
(full assistant text)

---

### Turn 2

**User:**
(verbatim)

**Assistant:**
(verbatim)

---

(and so on)
```

### Special cases

- **Skill invocation**: user message appears as `/skill-name [args]` (compressed by the script; original SKILL.md injection content is not preserved)
- **Local command**: user message appears as `[local: /command-name]` (e.g. `/compact`, `/clear`)
- **No assistant reply**: `**Assistant:** [no reply]`

---

## Sentiment values

| Value | Meaning | extract weight |
|-------|---------|----------------|
| `neutral` | Default, untagged | ×1.0 |
| `positive` | Good experience | ×1.0 |
| `negative` | Session with corrections | ×1.5 |

Set at record time via `--positive` / `--negative` flag; defaults to `neutral`. An optional `--reason "<text>"` can be added — it is written to the `reason` frontmatter field (omitted when not provided).

---

## Suggested tags

Common tags (extendable):

Task type: `debug` `design` `refactor` `research` `explain` `write` `review`

Output type: `code` `decision` `analysis` `template` `script`

Pattern tags (used in extract analysis):
- `repeat-constraint`: user repeatedly stated the same constraint
- `correction`: user corrected the AI's direction
- `workflow`: involves a fixed workflow
- `style-pref`: involves writing or response style preferences
